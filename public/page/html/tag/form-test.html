<form> 태그 테스트

enter 입력으로 submit이 발동되는 조건

#1: form + input[name=q] + button[type=submit]

#2: form + input + button[type=button]

#3: form + input[name=q] + button[type=button]

#4: form + input[name=q] + input[type=submit]

#5: (이건 안됨) no form + input[name=q] + input[type=submit]

#6: form without attribute + input[name=q] + input[type=submit]

#7: form with method + input[name=q] + input[type=submit]

#8: form without attribute + input[name=q]

설명

여기서 안되는건 <form> 태그가 없는 #5번 뿐이다. 즉, <input> 태그를 <form> 태그로 감싸주기만 하면 어떤 조건에서든 submit이 발생함.

심지어 버튼이 없어도!

당연히 이벤트 핸들러가 있는 경우는 제외다. 아래에 설명함.

Prevent submit

#1: 인라인으로 return false

#2: 인라인으로 event.preventDefault

#3: submit 핸들러에서 event.preventDefault()

#4: submit 핸들러에서 return false

설명

onsubmit 핸들러에서 false를 반환하거나, event.preventDefault()를 호출하면 막을 수 있음.
